Deligatr Assistant — test harness

Configure a mock GHL context, then load the widget — a chat bubble appears bottom-right, same as it will in a real sub-account. Serve this folder with python3 -m http.server — opening via file:// can break fetch in some browsers.

URL parsing test: leave the mock locationId blank and open test.html?locationId=ABC123&mock=… — the app must pick ABC123 from the URL (check the [dgtr] send console line).
Round-trip test: click the bubble, ask “how do I add a team member?”, confirm a real reply renders.
Plan nudge: set tier for your mock location to DeliHub Free in Supabase, ask for a 3rd user, expect the DeliHub Pro nudge.
Escalation: ask something mentioning “human” or “team” — the reply should carry offer_call: true, showing a small “Talk to a human” button under that message; clicking it opens booking_url in a new tab (no extra webhook call). Verify the real backend also fires the Slack message + GHL task for this case.
To use a local mock endpoint instead of the interceptor, paste its URL (e.g. http://localhost:5678/webhook/deligatr-assistant) into the endpoint field.
